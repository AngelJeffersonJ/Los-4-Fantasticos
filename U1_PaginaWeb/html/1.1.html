Programación Web

La programación web se refiere al proceso de escribir, codificar y construir las instrucciones que permiten que los sitios web y las aplicaciones web funcionen correctamente. Involucra el desarrollo de código para el lado del cliente (frontend) y el lado del servidor (backend) con el objetivo de crear aplicaciones web interactivas y funcionales.

1.1 Evolución de las aplicaciones web

La evolución de las aplicaciones web ha sido significativa a lo largo de los años, marcada por avances tecnológicos, cambios en las prácticas de desarrollo y la creciente demanda de experiencias más ricas y dinámicas en línea.

La evolución se fue dando de la siguiente manera:

Web Estática (Década de 1990)

En sus inicios, las aplicaciones web eran principalmente estáticas, compuestas principalmente por documentos HTML que se cargaban en el navegador. Los sitios web eran simples y no ofrecían mucha interactividad.

Web Dinámica (Finales de la Década de 1990 - Principios de la Década de 2000):

Con la introducción de tecnologías como CGI (Common Gateway Interface) y el uso extendido de servidores y bases de datos, las aplicaciones web empezaron a volverse más dinámicas. Las interacciones del usuario podían generar respuestas personalizadas del servidor.

AJAX y Web 2.0 (Principios de la Década de 2000):

La introducción de AJAX (Asynchronous JavaScript and XML) permitió actualizaciones de contenido asíncronas sin recargar toda la página. Esto llevó al surgimiento de la llamada "Web 2.0", donde las aplicaciones web comenzaron a ser más interactivas y colaborativas. Surgieron servicios web como Google Maps y Gmail, que aprovechaban estas tecnologías.

Frameworks del Lado del Cliente (Mediados de la Década de 2000 - Actualidad):

La popularidad de bibliotecas y frameworks del lado del cliente como jQuery, Angular, React y Vue.js permitió a los desarrolladores crear interfaces de usuario más complejas y eficientes. Estos frameworks facilitaron el desarrollo de aplicaciones de una sola página (SPA), donde la interfaz se actualiza dinámicamente sin recargar la página completa.

Arquitecturas del Lado del Servidor (Actualidad):

Se han desarrollado arquitecturas del lado del servidor más eficientes y escalables, como el enfoque de microservicios. Esto ha permitido la construcción de aplicaciones web más modulares y fácilmente mantenibles.

Aplicaciones Progresivas (PWA) (Década de 2010 - Actualidad):

Las PWA son aplicaciones web que ofrecen una experiencia similar a la de las aplicaciones nativas, incluyendo funcionalidades como notificaciones push, acceso sin conexión y actualizaciones automáticas. Esto ha mejorado la experiencia del usuario y ha permitido un mayor rendimiento en dispositivos móviles.

Arquitecturas sin Servidor (Serverless) (Actualidad):

La arquitectura sin servidor ha ganado popularidad, permitiendo a los desarrolladores construir y desplegar aplicaciones sin preocuparse por la infraestructura subyacente. Esto simplifica el desarrollo y la gestión de aplicaciones web.

Estos fueron unos de los ejemplos de los avances mas importantes que han hecho evolucionar a la programación web haciendo que cada vez sea más útil y eficiente.

1.2 Arquitectura de las aplicaciones web

La arquitectura de las aplicaciones web se refiere a la estructura y organización de los componentes que componen una aplicación web, así como a la forma en que interactúan entre sí.

La arquitectura se compone de 3 componentes principales:

  1. Un servidor Web
  2. Un servidor de base de datos
  3. Uno o más clientes (navegador web)

El servidor Web distribuye páginas de información formateada a los clientes que las solicitan. Los requerimientos son hechos a través de una conexión de red, y para ello se usa el protocolo HTTP. Una vez que se solicita esta petición mediante el protocolo HTTP y la recibe el servidor Web, éste localiza la página Web en su sistema de archivos y la envía de vuelta al navegador que la solicitó.

Las aplicaciones Web están basadas en el modelo Cliente/Servidor que gestionan servidores web, y que utilizan como interfaz páginas web.

Las páginas Web son el componente principal de una aplicación o sitio Web. Los browsers piden páginas (almacenadas o creadas dinámicamente) con información a los servidores Web. En algunos ambientes de desarrollo de aplicaciones Web, las páginas contienen código HTML y scripts dinámicos, que son ejecutados por el servidor antes de entregar la página.

Una vez que se entrega una página, la conexión entre el browser y el servidor Web se rompe, es decir que la lógica del negocio en el servidor solamente se activa por la ejecución de los scripts de las páginas solicitadas por el browser (en el servidor, no en el cliente). Cuando el browser ejecuta un script en el cliente, éste no tiene acceso directo a los recursos del servidor. Hay otros componentes que no son scripts, como los applets (una aplicación especial que se ejecuta dentro de un navegador) o los componentes ActiveX. Los scripts del cliente son por lo general código JavaScript o VBSscript, mezclados con código HTML.
